Especificaciones Técnicas — Asistente de Escritura Académica
Versión 1.0 (alineada al código real) — Junio 2026

================================================================================
1. ARQUITECTURA GENERAL
================================================================================

Stack:
- Backend: Node.js + Express (backend_academico.js), puerto 4000
- Frontend: SPA en public/app_frontend.html (HTML/CSS/JS vanilla)
- Persistencia local: JSON en data/sessions/*.json
- Configuración: variables en .env (proveedor LLM, keys, Modo Búnker)
- Despliegue opcional: Docker (docker-compose.yml) con bind 127.0.0.1:4000

Módulos:
- sessions_store.js — sesiones, capítulos, medidor de gasto (usage)
- guardian.js — clasificación de intención (regex + LLM rápido), estimación de costo
- audit_json.js — auditoría estructurada JSON, tarjetas aceptar/rechazar
- biblioteca_store.js — RAG BM25 local sobre ./biblioteca/
- docx_export.js — export Word a ./entregables/
- bunker.js — Modo Búnker (solo localhost, sin salida a internet)

================================================================================
2. GUARDIÁN DE INTENCIÓN (anti-evasión)
================================================================================

Flujo:
1. Regex sobre texto normalizado (leet-speak, patrones EN/ES de evasión)
2. Si ambiguo: llamada LLM con modelo económico (LLM_FAST_MODELS_JSON)
3. Si intención = evasión → bloqueo con mensaje pedagógico (sin llamar al modelo principal)

Archivo: guardian.js
- classifyIntentRegex(), parseGuardianJson(), estimateCostUsd(), estimateStageCostUsd()

================================================================================
3. PIPELINE DE ANÁLISIS (dos fases)
================================================================================

POST /api/suggest con stage:
- audit — Fase 1: JSON estructurado (tarjetas en UI)
- rewrite — Fase 2: texto optimizado + referencias
- full — ambas fases en una solicitud

Contexto inyectado al LLM:
- Norma (APA/MLA/Chicago/Vancouver) y nivel académico
- Memoria de sesión (últimos 4 capítulos)
- Academic Context: biblioteca local (BM25) + Crossref (si Modo Búnker OFF)

System Prompt v3.0: auditoría brutalmente honesta, sin reescritura en fase 1.

================================================================================
4. MODO BÚNKER (manifiesto soberano)
================================================================================

Variable: BUNKER_MODE=true en .env (también desde ⚙️ en frontend)

Cuando está activo:
- callLlmOnce() rechaza endpoints que no sean localhost/127.0.0.1
- searchAcademicData() (Crossref) devuelve vacío
- classifyIntentLLM() solo corre si el endpoint es local; si no, usa solo regex
- Badge en header: «0 bytes salen de esta máquina»

Uso típico: Ollama o LM Studio en http://localhost:11434

================================================================================
5. RED Y PRIVACIDAD
================================================================================

Servidor standalone:
- BIND_HOST=127.0.0.1 por defecto (solo esta máquina)

Docker:
- docker-compose.yml mapea 127.0.0.1:4000:4000 (no expone a la LAN)
- Dentro del contenedor: BIND_HOST=0.0.0.0 para aceptar el proxy de Docker

Badge 🔒 Fuente de datos (frontend):
«Tu texto se envía solo al proveedor LLM que VOS elegiste; nunca a nuestros servidores.»

Datos locales (no salen del disco salvo al LLM elegido):
- Sesiones, capítulos, índice biblioteca, entregables Word

================================================================================
6. MEDIDOR DE GASTO
================================================================================

Cada análisis exitoso acumula en session.usage:
- totalCostUsd (estimado por tokens/caracteres y tabla de precios en guardian.js)
- totalRequests, totalChars
- entries[] con stage (audit|rewrite|full), modelo, timestamp

Visible en:
- Panel de aprendizaje (barra superior)
- Lista de sesiones (resumen USD)

Confirmación de costo si estimateUsd > COST_CONFIRM_THRESHOLD_USD (default 0.10).

================================================================================
7. FRONTEND
================================================================================

Layout 3 columnas redimensionables:
- Sesiones + capítulos | Editor | Panel de aprendizaje

Funciones:
- Debounce → auditoría JSON → tarjetas → reescritura con sugerencias aceptadas
- Modo Tesis: chunking PDF/DOCX, pipeline pausable
- Export Word de sesión activa
- Tooltips en badges del header
- Configuración multi-proveedor LLM (banco de API keys)

================================================================================
8. API PRINCIPAL
================================================================================

GET  /api/config          — estado LLM, búnker, umbral de costo
POST /api/config          — guardar .env
POST /api/estimate        — estimación USD antes de analizar
POST /api/suggest         — auditoría / reescritura / full
GET  /api/sessions        — listar sesiones (con usage)
POST /api/sessions        — crear sesión
GET  /api/sessions/:id    — detalle
PATCH/DELETE sesiones y capítulos
POST /api/biblioteca/reindex
POST /api/export/docx
